# Daily News 产品需求文档

## 文档目的

定义当前仓库已经实现或明确约束的用户可见行为，帮助后续开发在不重新解读代码的情况下判断改动是否符合产品边界。本文仅根据当前仓库可见内容整理；未在仓库中找到证据的内容均不做假设。

## 适用范围

适用于修改 Daily News 前端展示、新闻内容规则、分类筛选、偏好设置、热点排序、搜索、可信度标签、数据 fallback 体验和中文化边界的工作。

不适用于数据库、生产部署、商业化、账号体系、通知系统或历史归档，因为当前仓库没有这些能力的证据。

## Plan 或项目证据

| 证据 | 可确认需求 |
| --- | --- |
| `README.md` | 项目是新闻日报原型；需要展示密集时间线、当前热点、分类筛选、可信度标签和偏好新闻 |
| `src/App.tsx` | 前端入口包含偏好新闻、分类导航、设置、搜索、热点、时间线、展开更多和手动刷新 |
| `src/types.ts` | 新闻报告包含 `generatedAt`、`items`、`sourceCount`、`notes`；新闻项包含 `primaryCategory`、`trust` 和评分拆解 |
| `src/lib/newsPipeline.ts` | 只展示启用来源，排序后过滤 `trust.shouldShow === false` 的新闻 |
| `src/lib/newsOrdering.ts` | 偏好列表按偏好类别筛选后按时间排序；热点列表不使用用户偏好分 |
| `AGENTS.md` | 用户侧页面 copy、新闻标题、摘要、来源名应保持中文；分类页使用 `primaryCategory` |

## 产品目标

Daily News 要解决的问题是：把多来源新闻整理成一个可浏览的日报界面，让用户能先看偏好相关新闻，也能按分类查看，并通过热点、评分原因和可信度标签快速判断哪些新闻值得优先阅读。

当前可确认目标：

| 目标 | 说明 |
| --- | --- |
| 多来源聚合 | 从 `src/config/sources.ts` 中启用的来源抓取或读取新闻 |
| 中文阅读体验 | 面向用户的页面文案、新闻标题、摘要和来源名尽量保持中文 |
| 可解释排序 | 每条新闻显示综合排序分和排序原因 |
| 信任提示 | 每条新闻显示高/中/低可信和可信度原因 |
| 分类不重复 | 每条新闻只属于一个 `primaryCategory` 分类页，辅助 `categories` 只用于解释 |
| 偏好控制 | 用户可在浏览器本地调整分类偏好和地区语言偏好 |

## 用户和使用场景

| 用户或角色 | 当前证据 | 使用场景 |
| --- | --- | --- |
| 新闻阅读者 | `src/App.tsx` 的导航、搜索、热点和偏好设置 | 打开网页查看当前偏好新闻、某个分类、当前热点或搜索指定话题 |
| 本地维护者 | `README.md`, `docs/runbook.md`, npm scripts | 配置来源、生成静态日报、启动本地 API、验证排序和展示 |
| 来源维护者 | `src/config/sources.ts` | 新增、禁用或调整来源、栏目、查询词、主分类和可信度 |

目标用户是否是单人自用、内部工具还是公开产品：待确认。

## 功能需求

| 编号 | 需求 | 验收方式 |
| --- | --- | --- |
| PRD-1 | 应优先从 `GET /api/news` 加载新闻报告，失败后加载 `/daily-news.json`，再失败后使用 `src/data/firecrawlSnapshot.ts` | 断开 API 时页面仍有兜底新闻 |
| PRD-2 | 页面必须提供“偏好新闻”、十个分类入口和“设置”入口 | 检查 `src/App.tsx` 导航和页面可见文本 |
| PRD-3 | 分类页必须按 `primaryCategory` 筛选，不能因为辅助 `categories` 让同一条新闻出现在多个分类页 | 修改分类逻辑时跑相关测试或人工检查 |
| PRD-4 | 偏好新闻必须只包含用户设为中/高偏好的类别，并按发布时间新到旧排序 | `src/lib/newsOrdering.test.ts` 通过 |
| PRD-5 | 当前热点必须按公共重要性、时效、来源可信度和内容质量排序，不受用户偏好分影响 | `src/lib/newsOrdering.test.ts` 通过 |
| PRD-6 | 页面必须支持搜索标题、摘要、来源、分类和可信度标签 | 在页面搜索框输入关键词能过滤列表 |
| PRD-7 | 每条新闻必须显示来源、综合分、可信度标签、标题、摘要、主分类、辅助标签、可信度原因和排序原因 | 检查 `NewsCard` 展示字段 |
| PRD-8 | 用户偏好保存在浏览器 `localStorage`，刷新页面后保留 | 修改偏好后刷新页面仍保留设置 |
| PRD-9 | 极低质量或缺少标题/链接的新闻不应展示 | `src/lib/scoring.test.ts` 中 invalid story 用例通过 |
| PRD-10 | 用户侧中文文案、新闻标题、摘要和来源名应保持中文；英文外部来源名至少通过 `sourceLabel` 兜底映射 | 页面检查和保存快照检查 |

## 非功能需求

| 维度 | 要求 |
| --- | --- |
| 可维护性 | 新来源和分类规则应优先在 `src/config/sources.ts` 和相关类型中显式配置 |
| 可解释性 | 排序和可信度必须保留人可读原因，不能只给分数 |
| 降级能力 | API、Firecrawl、直接来源抓取或翻译不可用时，应尽量保留静态/快照可读内容 |
| 数据最小化 | 浏览器不接触 `.env`、`.env.local` 或翻译密钥 |
| 本地优先 | 当前证据只支持本地开发和生产式本地运行 |

## 状态和边界情况

| 状态 | 当前行为 |
| --- | --- |
| API 正常 | 前端使用 `/api/news` 的 `DailyNewsReport` |
| API 不可用 | 前端加载 `/daily-news.json`，再兜底 `firecrawlSnapshotNews` |
| 加载中 | 刷新按钮显示旋转状态 |
| 加载失败 | 页面显示“实时接口暂不可用，当前显示本地兜底数据。” |
| 无搜索结果 | 页面显示“没有匹配当前筛选的新闻。” |
| 新闻多于首屏数量 | 通过“展开更多”每次增加 18 条 |
| 英文来源未翻译 | 没有翻译配置时，英文-only 结果可能被跳过；已进入页面的英文来源名由 `sourceLabel` 兜底 |

## 非目标

- 不实现账号、登录、权限、跨设备同步或服务端保存偏好。
- 不实现数据库、历史日报归档或长期检索。
- 不绕过付费墙；付费墙来源只使用公开标题、导语和元数据。
- 不把 Firecrawl 或翻译密钥暴露给浏览器。
- 不把 `public/daily-news.json` 当作编辑源数据。
- 不把社交媒体单点消息伪装成高可信新闻。

## 实现指引

- 改新闻来源、查询词、栏目、主分类、可信度或启用状态时，优先修改 `src/config/sources.ts`。
- 改排序权重和理由时，检查 `src/config/scoring.ts`、`src/lib/scoring.ts` 和 `src/lib/scoring.test.ts`。
- 改可信度策略时，检查 `src/lib/trust.ts` 和过滤逻辑 `trust.shouldShow`。
- 改分类页行为时，保持 `primaryCategory` 是唯一分类页入口；`categories` 只做标签解释。
- 改 UI 文案时，保持中文；如果外部来源名可能为英文，保留或扩展 `src/App.tsx` 的 `sourceLabel` 兜底。
- 改 fallback 体验时，保持 `/api/news` -> `/daily-news.json` -> `firecrawlSnapshotNews` 的用户可读降级链。

## 验收标准

- `npm test` 通过。
- 前端或 TypeScript 改动后 `npm run build` 通过。
- 页面能打开并展示新闻、热点、分类、搜索、可信度标签和偏好设置。
- API 不可用时页面仍显示静态或快照新闻。
- 用户侧新增文案和保存的新闻内容为中文，或有明确翻译/兜底策略。
